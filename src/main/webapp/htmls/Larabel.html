Introducción

Laravel es el framework PHP más adecuado tanto para principiantes como para programadores avanzados. Acelere el desarrollo de aplicaciones web y llévelas al mercado con métodos modernos de PHP orientados a objetos. La sintaxis expresiva y las características modernas atraerán a los desarrolladores que quieran construir aplicaciones robustas. Los marcos facilitan el proceso de desarrollo al vincular diferentes módulos entre sí. El marco proporciona un punto de partida para la creación de aplicaciones y permite a los desarrolladores centrarse más en el desarrollo. Laravel ofrece potentes características como una capa de base de datos de representación abstracta y una inyección de datos fiable y escalable.[1]
Además, incorpora algunas características de tecnologías como ASP.NET, MVC, CodeIgniter, Ruby on Rails, etc. Por lo tanto, todas sus características aumentarán el ritmo de desarrollo de la web. Si está familiarizado con los conceptos básicos de PHP y algunos scripts PHP avanzados, Laravel scripts PHP, Laravel le hará la vida más fácil. Laravel sigue el patrón arquitectónico Modelo-Vista-Controlador (MVC).[2]

MVC

MVC es una arquitectura de software que separa la lógica del dominio/aplicación/negocio del resto de la interfaz de usuario. Separar la lógica del dominio/aplicación/negocio del resto de la interfaz de usuario. Lo hace mediante dividiendo la aplicación en tres partes: el modelo, la vista y el controlador.
El modelo gestiona el comportamiento y los datos subyacentes, ya que responde a las solicitudes de información emitiendo instrucciones a responder a las solicitudes de información con instrucciones para cambiar el estado de su información; y Estado de la información; y notificación a los observadores en los sistemas basados en eventos cuando la información cambia, cualquier número de estructuras de datos o sistemas de almacenamiento. Es gestión decir son datos y aplicaciones. Las vistas representan los elementos de la interfaz de usuario de la aplicación.
elementos de la interfaz de usuario de la aplicación. Muestran los datos del modelo en un formato adecuado para las interfaces de usuario. Interfaces de usuario. El controlador recibe la entrada del usuario y llama a los objetos modelo y vista para realizar las acciones apropiadas.[2] Los objetos Modelo y Vista para realizar las acciones correspondientes.
Como la mayoría de los frameworks PHP, Laravel también tiene su estructura basada en el estándar MVC. Es un patrón de arquitectura que divide la aplicación en 3 capas:

  • M de model: que representa la capa de la base de datos;
  • V de view: que representa la capa de presentación, visible para los usuarios;
  • C de controller: que representa la regla del negocio, que es básicamente la lógica que tendrá tu aplicación con la base dedatos y la información que envía el usuario.

Además de la arquitectura MVC, Laravel facilita y acelera el desarrollo de aplicaciones. Aporta muchas soluciones ya preparadas, fáciles de configurar y adaptar a los problemas que todos los desarrolladores web sufren en el proceso de creación, entre ellas:

  • Ruta
  • Autentificación
  • Control de secciones
  • Manipulación de la base de datos
  • Validación.

¿Qué es laravel?

Laravel es el nombre de un framework desarrollado para trabajar con PHP. Fue desarrollado en 2011 por Taylor Otwell y con el tiempo ha ido ganando terreno a otros frameworks diseñados para trabajar con PHP, como Symfony o Zend Framework. Es un marco de desarrollo con una curva de aprendizaje muy rápida, por lo que maneja una sintaxis expresiva y elegante ya que pretende simplificar el desarrollo web , por ejemplo, Tareas de Punto Común. Ejemplo de desarrollo web tareas de punto común como la autenticación, el enrutamiento, las sesiones, el almacenamiento en caché, etc. Laravel proporciona un código abierto utilizado para el desarrollo de aplicaciones y servicios web utilizando PHP 5.Su objetivo es utilizar el código PHP para desarrollar nuevas aplicaciones de forma fácil y elegante. Fue creado y desarrollado en 2011 para mejorar la estabilidad y la influencia de los frameworks como Ruby on Rails, ASP.NET MVC y Sinatra.[4]
Es un marco que tiene un futuro potencial a pesar de su corta edad. Cuenta con un equipo activo y enérgico y presenta archivos atractivos con un contenido completo y fácil de entender. También proporciona la funcionalidad necesaria para desarrollar aplicaciones actuales de forma sencilla y segura. Resultó ofrecer datos interesantes como el enrutamiento RESTful, un glamuroso motor ligero o PHP nativo y otros recursos adicionales. Basado en varios elementos de Symfony, Laravel también proporciona una gran base para un código seguro y probado para sus aplicaciones web.[3]
Laravel también ofrece un conjunto de herramientas de última generación para interactuar con las bases de datos. Características de laravel

El framework Laravel trabaja con una avanzada arquitectura de carpetas, de manera que promueve la separación de archivos en un orden correcto y definido, que guiará a todos los miembros del equipo y será un estándar entre los diferentes proyectos. Por supuesto, también tiene una arquitectura de clases muy adecuada, que promueve la separación de código por responsabilidad. Su estilo arquitectónico es MVC. Contiene además un amplio conjunto de características, que sirven para realizar la mayoría de las aplicaciones web. Entre ellas, las que se denotan a continuación:
Un sistema de enrutamiento que facilita la creación y el mantenimiento de todo tipo de URL amigables y motores de búsqueda, rutas de API y más. Un sistema de abstracción de base de datos con un ORM potente pero fácil de usar que nos permite tratar los datos de la base de datos como si fueran objetos simples. Un sistema para crear colas de trabajo para que pueda ejecutar tareas en segundo plano y aumentar el rendimiento de la aplicación.Múltiples configuraciones para el envío de correos electrónicos con múltiples proveedores. Un sistema de notificación a los usuarios a través de correo electrónico, base de datos y otros canales Una abstracción del sistema de archivos que nos permite escribir datos en los proveedores de la nube y, por supuesto, en los discos del servidor con el mismo código. gestión de sesiones Sistema de autenticación con todo lo que necesita, como recordatorio de contraseña, verificación de cuenta, recordatorio de inicio de sesión de usuario, etc. La capacidad de acceder a los datos en tiempo real y recibir notificaciones cuando cambia en la base de datos.

¿Por qué es el framework preferido en la comunidad de desarrolladores?

El éxito de este framework se debe a varios factores, es fundamental la cantidad de herramientas o funcionalidades que nos ofrece, gracias a su pronto mantemiento y actualizaciones que ofrecen, agregando una cantidad de módulos a ayudado a que se incremente continuamente y hoy es posible que supere en varios factores a otras alternativas populares. Entre los puntos destacados son:
Facilidad de uso: la curva de aprendizaje es mucho más pronunciada. Esto ha permitido a los desarrolladores que ya utilizaban alternativas más sencillas para desarrollar aplicaciones y querían pasar a otros frameworks más completos.
Práctico: Laravel tiene un potente sistema de comandos de consola para realizar muchas tareas tediosas y repetitivas. Artisan, la herramienta de línea de comandos de Laravel, ha hecho que la experiencia de desarrollo de Laravel sea muy atractiva para quienes han elegido este framework.
Documentación y tutoriales: La documentación de Laravel siempre ha sido muy completa, pero también tiene un enfoque muy didáctico, por lo que se puede utilizar fácilmente como base de aprendizaje y no sólo como referencia.

Herramientas de laravel

Entra las herramientas mas utilizadas de lavarel están:
Forge: Es un sistema de gestión de servidores, como una especie de panel de control, con el que creas dominios, instalas Laravel, distribuyes aplicaciones, etc.
Vapor: Permite distribuir aplicaciones Laravel sin necesidad de un servidor. En otras palabras, es un backend "sin servidor".
Nova: Este es un sistema de administración de contenido que le permite crear paneles de administración típicos de una manera muy cómoda.
Envoyes: permite la integración continua de aplicaciones Laravel, con solo empujarla al repositorio y la aplicación se desplegará en la nueva versión sin interrumpir el servicio.
Spark: le permite crear aplicaciones basadas en código en el mismo Laravel, evitando muchas partes que pueden repetirse en muchos proyectos.
Otros servicios de lavarel, tanto como gratuito o de paga:
Homestead: Permite máquinas virtuales para facilitar el desarrollo de Laravel en las condiciones más profesionales.
Valet: Permite en Mac crear entornos de ejecución de Laravel para desarrollo muy ligeros y fáciles de configurar mediante comandos de consola.

Paquetes oficiales proporcionados por LARAVEL

Cajero

Es uno de los paquetes oficiales que proporciona una interfaz expresiva y elegante para colaborar en la facturación de suscripciones de Stripe y Braintree. Permite ayuda para todos los códigos de facturación de suscripciones recurrentes que no desee escribir. Puede administrar cupones, suscripciones de intercambio, montos de suscripción, períodos de notificación e incluso crear facturas en PDF.

Enviado

Enviado proporciona una sintaxis limpia y mínima para configurar tareas comunes para realizar en servidores remotos. Actualmente, Enviado solo es compatible con los sistemas operativos Mac y Linux.

Pasaporte

Laravel facilita la autenticación de API con Laravel Passport proporciona a su aplicación Laravel una implementación de servidor OAuth2 completa en muy poco tiempo. Passport se basa en el servidor League OAuth2 administrado por Alex Bilbie.

Explorador

Explorador proporciona una solución simple basada en controlador para agregar búsqueda de texto completo a sus modelos eloquent y sincronizar su índice de búsqueda con sus registros eloquent

Socialité

Socialité proporciona una interfaz potente y fluida para la autenticación OAuth con Facebook, Twitter, Google, LinkedIn, GitHub y Bitbucket [2].

Vista

Esta es la sección de código que se asigna a la pantalla del usuario. Una vista se puede definir así: Una representación visual de una aplicación web que permite a los usuarios interactuar directamente con la aplicación. Laravel también enfatiza el uso de esta vista en las plantillas. La creación de plantillas en Laravel se realiza en Blade [2]. Luego usamos Admin LTE Bootstrap para crear una mirada interesante a un sistema de información de gestión de laboratorio utilizando el marco de Laravel.

Controlador

Código que crea la conexión entre el modelo y la vista. La función del controlador es obtener la dirección del enrutamiento que maneja. Todos los controladores en una aplicación Laravel se encuentran en la carpeta app/Http/controller.

Blade

El motor de plantillas Blade limpia sus vistas al proporcionar reemplazos estéticamente agradables para PHP en línea e incluye nuevas funciones potentes. Y por esa razón necesitas agregar una extensión de 'blade.php’. Blade elimina las etiquetas de PHP y proporciona accesos directos para tareas comunes para que sus vistas sean más limpias y fáciles de crear y mantener [6].

Artisan

Laravel también viene con una herramienta de interfaz de línea de comandos llamada Artisan. Con Artisan es un desarrollador que puede interactuar con la aplicación que sirve para desencadenar acciones como ejecutar migraciones, ejecutar pruebas unitarias y ejecutar tareas programadas. Artisan también es completamente extensible para que pueda escribir cualquier tipo de funcionalidad que desee [7].

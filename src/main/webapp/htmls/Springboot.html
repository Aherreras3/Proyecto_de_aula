Spring Boot

Spring framework es un software de código abierto desarrollado por el Spring Source, este framework puede ser utilizado en contenedores web, dispensando utilizados en los contenedores web que prescinden de los servidores de aplicaciones servidores de aplicaciones como JEE (Java Enterprise Edition) y JBoss.
Aunque Spring Framework es muy potente, la configuración inicial del marco suele ser bastante tediosa. Dado que la configuración inicial del framework suele ser bastante tediosa, el equipo de Spring ha lanzado Spring Boot, lo que simplifica considerablemente este proceso.

Spring Framework

Este framework nos proporciona varias características, de las cuales podemos destacar la inyección de dependencias, la gestión de transacciones, el soporte para pruebas automatizadas y el soporte para aspectos orientados a la programación.
Spring surgió como una alternativa ligera al entonces Java Enterprise Edition (JEE, alejándose del desarrollo de componentes y ofreciendo un enfoque más sencillo del desarrollo. Spring ofrece un enfoque más sencillo para el desarrollo de Java empresarial.
Spring cuenta con varios frameworks complementarios que son:

  • Spring MVC: este es utilizado para el desarrollo de aplicaciones web (este es parte e spring framework).
  • Spring Security: Este es utilizado para la inserción de funcionalidades para la autenticación y autorización.
  • Spring Data: Destinado para aplicaciones que utilizan nuevas tecnologías de almacenamiento de datos como base de datos y servicios en la nube.
  • Spring web Services: Utilizado para la creación de servicios web basados en SOAP.
  • Spring web Flow: Este es una extensión de Spring MVC que permite la implementación de flujos (Wizards) entre pantallas.

Historia

Spring fue escrito por Rod Johnson y su desarrollo comenzó en el año 2000.
Dos años después, en 2002, se publicó la primera versión de Spring Framework (versión 0.9), creada por Rob Johnson. Y ese mismo año se publicó el famoso libro "Expert One-on-One J2EE Design and Development" de Rod Johnson. Rod tardó 9 meses en escribirlo.
Spring es un marco de aplicación desarrollado por Interface 21 para aplicaciones escritas en el lenguaje de programación Java. Fue creado gracias a la colaboración de grandes programadores, entre los que Rod Johnson y Jürgen Hõller son los principales actores y líderes de este proyecto. Estos dos desarrolladores, entre otros colaboradores, han reunido toda su experiencia en el desarrollo de aplicaciones J2EE (Java 2 Enterprise Editions), incluyendo EJB (Enterprise JavaBeans), incluyendo EJB (Enterprise JavaBeans), servlets y JSP (Java Server Pages), y han conseguido combinar estas y otras herramientas en un único paquete para proporcionar una estructura más sólida y un mejor soporte para este tipo de aplicaciones.
Este marco de trabajo está disponible actualmente en la versión 5.3.22, y aunque es una versión temprana, es cada vez más popular. Una de las características que contribuyen a este éxito es el hecho de que se trata de una aplicación de código abierto, lo que significa que no tiene ningún coste ni se requiere ninguna licencia para utilizarla. Además, todo el código fuente del framework se incluye en el paquete de instalación.
Spring no trata de "reinventar la rueda", sino de integrar las distintas tecnologías existentes en un marco para facilitar y hacer más eficiente el desarrollo de aplicaciones J2EE portátiles entre servidores de aplicaciones. En 2003, un año después de la publicación del libro, se formó un equipo de desarrolladores para ampliar el framework en Sourceforge.Y lanzar la versión 0.9 (no fue el lanzamiento). En 2004, apareció por fin la primera versión importante, la versión oficial Spring 1.0 con la licencia Apache 2.0.
Pronto será aceptado por un gran número de desarrolladores de Java, pero ese momento aún no ha llegado. En agosto de ese mismo año, Rod Johnson, Jürgen Hoeller, Keith Donald y Colin Sampaleanu fundan una empresa de consultoría llamada Interface21. En el verano de 2005, Spring Framework se había descargado 400.000 veces. En 2006, se publicó Spring 2.0 con el objetivo de simplificar los archivos de configuración XML y dar soporte a Java 5. En 2007, la consultora que había fundado Interface21 cambió su nombre por el de SpringSource.
También lanzó Spring 2.5, que introdujo nuevas funciones como las configuraciones de anotaciones. En 2009, SpringSource fue adquirida por Convent e Hyperic. Posteriormente, SpringSource fue adquirida por VMWare por la friolera de 420 millones de dólares. Ese mismo año se lanzó SpringSource 3.0, compatible con Java EE6. En 2012, se lanzó Spring 3.2, que introdujo la configuración de Java y añadió soporte para Java 7, Hibernate 4 y Servlet 3.0. También requería al menos la versión 1.5 de Java.

Arquitectura de Spring

Módulo principal de primavera

El módulo Spring Core, el componente principal del marco de trabajo de Spring, proporciona el contenedor IoC. Hay dos tipos de implementaciones de contenedores de Spring, a saber, la fábrica de frijoles y el contexto de aplicación. La fábrica de frijoles se implementa a través de la interfaz org.springframework.beans.factory.BeanFactory y actúa como contenedor de beans. El contenedor de la fábrica de frijoles permite desacoplar las dependencias de configuración y especificación de la lógica del programa. En el marco de trabajo de Spring, la fábrica de frijoles actúa como un contenedor IoC central responsable de crear instancias de objetos de aplicación. También configura y compone las dependencias entre estos objetos. Existen numerosas implementaciones de la interfaz BeanFactory. La clase XmlBeanFactory es la implementación más común de la interfaz BeanFactory.

Módulo Spring AOP

Al igual que la programación orientada a objetos (POO), que divide las aplicaciones en jerarquías de objetos, la POO divide los programas en aspectos o preocupaciones. El módulo AOP de Spring permite la implementación de concerns o aspectos en una aplicación Spring. Los aspectos son beans regulares de Spring o clases regulares anotadas con la anotación @Aspect. Estos aspectos ayudan a la gestión de las transacciones y al registro y seguimiento de los errores en una aplicación. La gestión de las transacciones es necesaria, por ejemplo, en las operaciones bancarias, como la transferencia de un importe de una cuenta a otra. Importe de una cuenta a otra. El módulo AOP de Spring proporciona una capa de abstracción de gestión de transacciones que puede aplicarse a las API.

Módulo Spring ORM

Spring ORM se utiliza para acceder a datos de bases de datos en una aplicación. Proporciona API para manipular bases de datos con JDO, Hibernate e iBatis. Spring ORM es compatible con DAO, que proporciona una forma conveniente de crear las siguientes soluciones ORM basadas en DAO:

  • Gestión simple de transacciones declarativas.
  • Manejo transparente de excepciones.
  • Clases de plantillas ligeras y seguras para subprocesos.
  • Clases de apoyo DAO.
  • Administracion de recursos.

Módulo Spring Web MVC

El módulo Web MVC de Spring implementa la arquitectura MVC para crear aplicaciones web. Separa el código del modelo y los componentes de la vista de una aplicación web. En Spring MVC, cuando una petición es generada por el navegador, primero va a la clase DispatcherServlet (controlador frontal), que envía la petición a un controlador (clase SimpleFormController o clase AbstractWizardformController) utilizando una serie de mapeos de controlador. El controlador extrae y procesa la información contenida en una solicitud y envía el resultado a la clase DispatcherServlet en forma de objeto modelo. Por último, la clase DispatcherServlet utiliza las clases ViewResolver para enviar los resultados a una vista que muestra estos resultados a los usuarios.

Módulo Spring Web MVC

Spring Web Flow es una extensión del módulo Spring Web MVC. El marco Spring Web MVC proporciona controladores de formularios, como la clase SimpleFormController y la clase AbstractWizardFormController, para implementar un flujo de trabajo predefinido. Spring Web Flow ayuda a definir el archivo XML o la clase Java que gestiona el flujo de trabajo entre las diferentes páginas de una aplicación web. Spring Web Flow se distribuye por separado y puede descargarse del sitio web http://www.springframework.org.
Las ventajas de Spring Web Flow se describen a continuación:

  • Las diferentes interfaces de usuario de la aplicación se define de forma única mediante la definición del flujo web en un archivo XML.
  • Las definiciones de Web Flow ayudan a dividir virtualmente una aplicación en diferentes módulos y a reutilizar estos módulos en diferentes situaciones.
  • El ciclo de vida de Spring Web Flow se puede gestionar automáticamente.

Módulo DAO de Spring Web

El paquete DAO en el marco de trabajo de Spring proporciona soporte DAO utilizando tecnologías de acceso a datos como JDBC, Hibernate o JDO. Este módulo introduce una capa de abstracción JDBC eliminando la necesidad de la tediosa codificación JDBC. También proporciona clases de gestión de transacciones programáticas y declarativas. El paquete DAO de Spring admite la conectividad heterogénea de bases de datos Java y la asignación O/R, lo que permite a Spring trabajar con diferentes tecnologías de acceso a datos. Para un acceso fácil y rápido a los recursos de la base de datos, el marco de trabajo de Spring proporciona clases base DAO abstractas. Para cada tecnología de acceso a datos soportada por el marco de trabajo de Spring, existen múltiples implementaciones. Para JDBC, por ejemplo, la clase JdbcDaoSupport y sus métodos se utilizan para acceder a la instancia de DataSource y a una instancia de JdbcTemplate preconfigurada.

Módulo de contexto de la aplicación Spring

El módulo de contexto de aplicación de Spring se basa en el módulo central. El contexto de aplicación org.springframework.context.ApplicationContext es una interfaz BeanFactory. Este módulo proviene del paquete org.springframework.beans y también soporta características como la internacionalización (I18N), la validación, la propagación de eventos y la carga de recursos. El contexto de aplicación implementa la interfaz MessageSource y proporciona funciones de mensajería a una aplicación.

Spring Boot

Spring Boot es la solución para evitar tener que configurar todas las aplicaciones basadas en Spring, por lo que se puede decir que el objetivo principal de Spring Boot es proporcionar las herramientas para construir aplicaciones de Spring de más simple y que sean fáciles de configurar. Este nos permite crear aplicaciones auto contenidas dejando a un lado la arquitectura y enfocarnos solamente en el desarrollo, dejando que Spring Boot se encargue de labores como configurar las dependencias, desplegar nuestro servicio o aplicación a un servidor de aplicaciones y enfocarnos solamente en crear nuestro código. Por lo mencionado anteriormente Spring Boot utiliza internamente un servidor de aplicaciones embebido, por defecto este utiliza Tomcat, aunque también lo podemos hacer con jetty o undertow. Además, Spring Boot proporciona un completo gestor de dependencias como maven o gradle, configuraciones automáticas y mucho más para facilitar al desarrollador y permitir que la aplicación quede a la medida.

Historia

En octubre de 2012, Mike Youngstrom creó una solicitud de función en Spring jira que requería soporte para una arquitectura de aplicación web sin contenedores en el framework de Spring. Habló de la configuración del servicio del contenedor web en el contenedor principal para controlar el contenedor de Spring. Este es un extracto de la solicitud de Jira. Creo que la arquitectura de las aplicaciones web de Spring puede simplificarse en gran medida proporcionando herramientas y arquitecturas de referencia que utilicen componentes de Spring y modelos de configuración descendentes. La configuración de estos servicios comunes del contenedor web debería estar incrustada y unificada en el contenedor de Spring, guiada por el sencillo método main (). Este requisito impulsó el desarrollo del proyecto Spring Boot, que comenzó a principios de 2013.
En abril de 2014, se publicó Spring Boot 1.0.0. Desde entonces, se han publicado varias versiones de Spring Boot:

  • Spring boot 1.1: (junio de 2014)- Soporte de plantilla mejorado, soporte de gemfire, configuración automática de elasticsearch y apache.
  • Spring boot 1.2: (marzo de 2015)-Actualizar a servlet 3.1 / tomcat 8 / jetty 9, actualización de primavera 4.1, soporte de anotación banner / jms / SpringBootApplication.
  • Spring boot 1.3: (diciembre de 2016)-Actualización de Spring 4.2, nuevas herramientas de arranque de primavera para la configuración automática de la tecnología de caché (ehcache, hazelcast, redis e infinispan) y soporte jar totalmente ejecutable.
  • Spring boot 1.4: (enero de 2017)-Actualización de Spring 4.3, soporte couchbase / neo4j, análisis de falla de inicio y RestTemplateBuilder.
  • Spring boot 1.5: (febrero de 2017)-Soporta kafka / ldap, actualización de la biblioteca de terceros, desprecia el soporte de CRaSH y el punto final del registrador ejecutor para modificar dinámicamente el nivel de registro de la aplicación.
  • Spring boot 2.0: marzo de 2018)-Basado en Java 8, compatible con Java 9, compatible con Quartz, el programador simplifica enormemente la configuración automática de seguridad, admite Netty incorporado.

La simplicidad de Spring Boot permite a los desarrolladores de Java adoptar el proyecto rápidamente y a gran escala. Spring Boot es posiblemente una de las formas más rápidas de desarrollar aplicaciones web de microservicios basadas en REST en Java.

Características de Spring Boot

Contenedor de aplicaciones integrado

Spring Boot nos permite compilar nuestras aplicaciones tanto como un archivo.jar como un archivo .war.
Un archivo .jar lo ejecuta como si se tratase de una aplicación Java normal.
Un archivo .war lo compila desplegándolo en un servidor de aplicaciones como lo es Tomcat.
Todo esto es posible y que Spring Boot tiene in servidor d aplicaciones integrado .jar y levantándose cuando ejecutamos la aplicación, de esta forma podemos distribuir nuestras aplicaciones de una forma más sencilla.

Starters

Spring Boot nos proporciona una gran serie de dependencia las cuales se les denominan Starters, las cuales se pueden añadir a nuestro proyecto dependiendo de lo que necesitemos: crear un controlador REST, acceder a una base de datos usando JDBC, etc.
Y una vez añadido el starter que necesitamos, este nos proporcionara todas las dependencias necesarias para nuestro proyecto, tanto de Spring, como de terceros. Además, dichos starters vienen configurados con valores por defecto que pretenden minimizar la necesidad de configuración.
Un ejemplo de un starter es spring-boot-starter-actuator: el cual una vez añadido, nuestra apliacion empezara a geberar métricas tanto de la JVM (Java Virtual Machine) como de la aplicación en sí (latencias, errores, etc).

Spring Initializr

Esta herramienta fue creada por el equipo de Spring con el fin de generar de una manera más intuitiva el esqueleto de la aplicación que se desea desarrollar.
Esta herramienta genera el pom.xml, el main de la aplicación y realiza un test que se encarga de comprobar que la aplicación arranca de manera correcta.
Además, de que de esta forma, en lugar de tener que elegir 10 o 20 dependencias es mucho más cómodo elegir 2 starters y dejar que Spring Boot se encargue del resto.
Después de crear el esqueleto de la aplicación solo basta con importarlo a un IDE que permita el trabajo con Java, se recomienda Eclipse o Apache Netbeans.

  • Permite la creación de rutas dinámicas tanto para las vistas como para los entornos que lo requieran.
  • Trabaja perfectamente con el paradigma MVC (Modelo Vista Controlador).
  • Soporte para el Desarrollo de aplicaciones Reactivas con Spring WebFlux (Tipo React JS y similares) y para la creación y consumo de APIs desde dispositivos móviles, Smart TV’s, etc.

Anotaciones principales Spring Framework

A continuación, pasaremos a describir algunas de las principales notaciones utilizadas en Spring Fremework.

@Required

Esta anotación se aplica principalmente al método “setters” de beans. Esta notación dará a entender que una propiedad es requerida, en caso contrario se genera una excepción.

@Autowired

Esta anotación inyecta la dependencia del objeto implícitamente, básicamente podemos decir que esta notación crea los lazos entre los distintos elementos.

@Qualifier

Esta anotación se usa junto con @Autowired. Esta notación se utiliza cuando se tiene más de un vean del mismo tipo y se desea conectar solo con uno de ellos con una propiedad.

@Configuration

Una clase Java anotada con @Configuration es una configuración en sí misma y tendrá métodos para crear instancias y configurar las dependencias. 

                
                     @Configuration
                       public class DataConfig {
                       @Bean
                       public DataSource source() {
                       DataSource source = new OracleDataSource();
                       source.setURL();
                       source.setUser();
                       return source;
                       }
                       @Bean
                       public PlatformTransactionManager manager() {
                       PlatformTransactionManager manager = new BasicDataSourceTransactionManager();
                       manager.setDataSource(source());
                       return manager;
                       }
                       }

Paquetes que ofrece Spring Boot

  • Paquete controller para todos los endpoints que tenga nuestra aplicación.
  • Paquete service para agregar alli las clases que respondan a la funcionalidad y lógica.
  • Paquete model para las representaciones de nuestro modelo de datos (entidades).
  • Paquete repository para las clases que establecen la comunicación con la base de datos.

Ventajas y Desventajas de Sprint boot

Ventajas

  • El framework Spring es muy versátil pues se divide en módulos diseñados para solventar diferentes necesidades. Por ejemplo, el módulo Spring-Core nos permite reducir líneas de Código.
  • Permite el desarrollo de aplicaciones flexibles y escalables; es decir, a medida que crezca el programa, su mantenimiento siempre será fácil pues el código será fácil de entender.
  • En resumen, el framework spring permite al desarrollador enfocarse en el proceso de la aplicación, pues además de facilitarle la gestión de base de datos, su gran cantidad de librerías disponibles le ayuda a tener flexibilidad.

Desventajas

  • Es demasiado complejo.
  • Funcionalidades a lo mejor no funcionan como tú quieres o como tú crees que están funcionando.
